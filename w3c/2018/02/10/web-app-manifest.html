Reading "Web App Manifest"

Feb 10, 2018

Last time update: 2018-02-10

Overview

This is my memo of reading w3c spec “Web App Manifest”. At first, at the time I am writing this, status of the spec is still Working Draft which means it could be changed later on. In case you want to get more involved to see what is going on about the specification in real time here is the “github”.

What is Web App Manifest

It provides a standard of metadata for web application so that UA (web browser mostly) could handle those meta information properly. This might sounds too abstract to understand. Here is a very simple web app using manifest. For this tiny example we only need index.html, manifest.json and an icon file.

This Github Sample is by Jeffxz
examples/web-app/manifest-sample1/index.html view raw
This Github Sample is by Jeffxz
examples/web-app/manifest-sample1/manifest.json view raw

Here is some explain about this simple app. index.html is still entry of the web app, just like other normal web site. There is a link element pointing to manifest.json with ref="manifest" to let UA know there is a manifest file corresponding with this web app.

Unfortunately (maybe not) UA’s behavior of treating manifest is different from each other so I used latest chrome (v63.0.3239.132) on windows 10 to show how this works. Following is the setting menu in chrome to add this web app to desktop.

Then you will see a popup window as following which used the icons and name field from your manifest.json file.

After this you will find there is an icon on your windows desktop for this tiny web app.

Click it you will see the web app started just like other windows applications.

Tada!!! Here is your very first tiny web app launched.
